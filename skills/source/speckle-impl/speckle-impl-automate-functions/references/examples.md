# Working Code Examples (Speckle Automate Functions)

## Example 1: Complete Python Function — Forbidden Type Checker

This example shows the full path from template to deployable function. It checks a model for objects of a forbidden speckle_type and reports errors.

### flatten.py

```python
from specklepy.objects import Base
from typing import Iterable


def flatten_base(base: Base) -> Iterable[Base]:
    """Recursively flatten a Base object hierarchy."""
    elements = getattr(base, "elements", getattr(base, "@elements", None))
    if elements is not None:
        for element in elements:
            yield from flatten_base(element)
    yield base
```

### main.py

```python
from pydantic import Field, SecretStr
from speckle_automate import (
    AutomateBase,
    AutomationContext,
    execute_automate_function,
)
from flatten import flatten_base


class FunctionInputs(AutomateBase):
    whisper_message: SecretStr = Field(
        title="Secret Message",
        description="A secret value that will not appear in logs.",
    )
    forbidden_speckle_type: str = Field(
        title="Forbidden Speckle Type",
        description=(
            "If an object has this speckle_type,"
            " it will be marked with an error."
        ),
    )


def automate_function(
    automate_context: AutomationContext,
    function_inputs: FunctionInputs,
) -> None:
    # 1. Receive the version data
    base = automate_context.receive_version()

    # 2. Flatten and filter objects
    objects = list(flatten_base(base))
    forbidden = [
        obj for obj in objects
        if obj.speckle_type == function_inputs.forbidden_speckle_type
    ]

    # 3. Report results
    if forbidden:
        automate_context.attach_error_to_objects(
            category="Forbidden Type",
            object_ids=[obj.id for obj in forbidden],
            message=f"Found {len(forbidden)} objects of forbidden type "
                    f"'{function_inputs.forbidden_speckle_type}'.",
        )
        automate_context.mark_run_failed(
            f"{len(forbidden)} objects of forbidden type found."
        )
    else:
        automate_context.set_context_view()
        automate_context.mark_run_success("No forbidden objects found.")


# Entry point — MUST be at module level
if __name__ == "__main__":
    execute_automate_function(automate_function, FunctionInputs)
```

## Example 2: Complete C# Function — Object Count Validator

### FunctionInputs.cs

```csharp
using System.ComponentModel;
using System.ComponentModel.DataAnnotations;

public readonly struct FunctionInputs
{
    [Required]
    public string SpeckleTypeToCount { get; init; }

    [DefaultValue(10)]
    [Range(1, 100)]
    [Required]
    public int SpeckleTypeTargetCount { get; init; }

    [Required]
    [Secret]
    public string ExternalServiceKey { get; init; }
}
```

### AutomateFunction.cs

```csharp
using Speckle.Automate.Sdk;
using Speckle.Sdk.Models.Extensions;

public class AutomateFunction
{
    public async Task Run(
        IAutomationContext automationContext,
        FunctionInputs functionInputs)
    {
        // 1. Receive the version data
        var rootObject = await automationContext.ReceiveVersion();

        // 2. Flatten and filter
        var allObjects = rootObject.Flatten().ToList();
        var matchingObjects = allObjects
            .Where(o => o.speckle_type == functionInputs.SpeckleTypeToCount)
            .ToList();

        // 3. Report results
        if (matchingObjects.Count != functionInputs.SpeckleTypeTargetCount)
        {
            automationContext.AttachErrorToObjects(
                "Count Mismatch",
                matchingObjects.Select(o => o.id).ToList(),
                $"Expected {functionInputs.SpeckleTypeTargetCount} objects "
                + $"of type '{functionInputs.SpeckleTypeToCount}', "
                + $"but found {matchingObjects.Count}."
            );
            automationContext.MarkRunFailed(
                $"Expected {functionInputs.SpeckleTypeTargetCount} "
                + $"but found {matchingObjects.Count}."
            );
        }
        else
        {
            automationContext.MarkRunSuccess(
                $"Found exactly {matchingObjects.Count} matching objects."
            );
        }
    }
}
```

### Program.cs


## Example 3: Python Function with File Artifact Output

```python
import csv
import io
from pydantic import Field
from speckle_automate import (
    AutomateBase,
    AutomationContext,
    execute_automate_function,
)
from flatten import flatten_base


class FunctionInputs(AutomateBase):
    target_speckle_type: str = Field(
        title="Target Speckle Type",
        description="The speckle_type to audit and export to CSV.",
    )


def automate_function(
    automate_context: AutomationContext,
    function_inputs: FunctionInputs,
) -> None:
    base = automate_context.receive_version()

    # Collect matching objects
    matching = [
        obj for obj in flatten_base(base)
        if obj.speckle_type == function_inputs.target_speckle_type
    ]

    # Generate CSV report
    report_path = "audit_report.csv"
    with open(report_path, "w", newline="") as f:
        writer = csv.writer(f)
        writer.writerow(["Object ID", "Speckle Type"])
        for obj in matching:
            writer.writerow([obj.id, obj.speckle_type])

    # Attach file artifact to run results
    automate_context.store_file_result(report_path)

    # Report outcome
    automate_context.mark_run_success(
        f"Exported {len(matching)} objects to CSV report."
    )


if __name__ == "__main__":
    execute_automate_function(automate_function, FunctionInputs)
```

## Example 4: Python Function with Category-Based Error Grouping

```python
from pydantic import Field
from speckle_automate import (
    AutomateBase,
    AutomationContext,
    execute_automate_function,
)
from flatten import flatten_base


class FunctionInputs(AutomateBase):
    required_property: str = Field(
        title="Required Property",
        description="Property name that MUST exist on all objects.",
    )
    strict_mode: bool = Field(
        default=False,
        title="Strict Mode",
        description="When true, treat warnings as errors.",
    )


def automate_function(
    automate_context: AutomationContext,
    function_inputs: FunctionInputs,
) -> None:
    base = automate_context.receive_version()
    objects = list(flatten_base(base))

    # Check for missing required property
    missing_prop = [
        obj for obj in objects
        if not hasattr(obj, function_inputs.required_property)
    ]

    # Check for empty values
    empty_value = [
        obj for obj in objects
        if hasattr(obj, function_inputs.required_property)
        and not getattr(obj, function_inputs.required_property)
    ]

    has_errors = False

    # Attach errors by category
    if missing_prop:
        automate_context.attach_error_to_objects(
            category="Missing Property",
            object_ids=[obj.id for obj in missing_prop],
            message=f"Property '{function_inputs.required_property}' is missing.",
        )
        has_errors = True

    if empty_value:
        automate_context.attach_error_to_objects(
            category="Empty Value",
            object_ids=[obj.id for obj in empty_value],
            message=f"Property '{function_inputs.required_property}' is empty.",
        )
        if function_inputs.strict_mode:
            has_errors = True

    # Final outcome
    if has_errors:
        total = len(missing_prop) + len(empty_value)
        automate_context.mark_run_failed(
            f"{total} objects have property issues."
        )
    else:
        automate_context.set_context_view()
        automate_context.mark_run_success("All objects have the required property.")


if __name__ == "__main__":
    execute_automate_function(automate_function, FunctionInputs)
```

## Example 5: End-to-End Deployment Workflow

This is NOT code but the exact sequence of steps to go from zero to a published function.

### Step 1: Create Function via Wizard

1. Open your Speckle Enterprise project
2. Navigate to the **Automations** tab
3. Click **View Functions** then **New Function**
4. Authorize GitHub OAuth (if prompted)
5. Select **Python** template
6. Fill in metadata: Name, Description, Tags
7. Select your GitHub organization
8. Click **Create** — wizard clones the template repo

### Step 2: Modify Function Code

```bash
# Clone the auto-created repo
git clone https://github.com/your-org/your-function-repo.git
cd your-function-repo

# Edit main.py with your validation logic
# Edit pyproject.toml if you need additional dependencies
```

### Step 3: Test Locally

```bash
pip install -e ".[dev]"
pytest tests/
```

### Step 4: Push and Release

```bash
git add .
git commit -m "feat: implement compliance checker"
git push origin main

# Create a release on GitHub (via UI or CLI)
gh release create v1.0.0 --title "v1.0.0" --notes "Initial release"
```

### Step 5: Verify Deployment

1. The GitHub Actions workflow triggers automatically on release
2. Wait for the workflow to complete (green checkmark)
3. Navigate to the Function Library in Speckle
4. Verify your function appears with the correct version

### Step 6: Create an Automation

1. Go to any project in your workspace
2. Navigate to the **Automations** tab
3. Click **New Automation**
4. Select your function from the Function Library
5. Choose the target model
6. Configure input parameters
7. Save — the automation now triggers on every new version
